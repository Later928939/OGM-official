1. システム全体像 ― 何がつながっているか

OGM公式LINEBotは、以下の5つのサービスが連携して動いています。ユーザーがLINEでメッセージを送ると、それが自動的に処理されてさまざまな機能が実行されます。

簡単に言うと： ユーザーがLINEにメッセージを送る → Railway上で動くプログラム（index.js）が受け取り → 内容に応じてAI応答・登録処理・告知配信などを実行する、という流れです。

3. ユーザー向け機能 ― LINEで何ができるか

📝 メンバー登録機能

「登録」「参加登録」「登録する」のいずれかを送ると、3ステップでメンバー登録できます。

「登録」と送信
→
学籍番号を入力
（例: k24001234）
→
氏名を入力
→
メールアドレスを入力
→
✅ 登録完了！
Sheetsに保存
学籍番号は「英字1文字＋数字6〜8桁」の形式のみ受け付けます（例: k24001234）。
🤖 AI自動応答機能

登録フロー中でなく、コマンドでもないメッセージを送ると、Claude AIがSYSTEM_PROMPTに書かれたOGMの情報を参照しながら自動で返答します。イベントの詳細・OGMについての質問などに答えられます。

👋 フォロー時の自動挨拶

ユーザーがLINEアカウントを友だち追加したとき、設定されたGREETING_MESSAGEが自動送信されます。

✅ イベント参加ボタン（ポストバック）

管理者がボタン付きの告知を配信した際、ユーザーが「参加する」を押すと、参加記録がGoogle Sheetsに自動保存されます。

4. 管理者向けコマンド一覧

管理者のLINEアカウントからのみ使えるコマンドです。LINEチャット画面でそのまま打ち込みます。

/告知          → SYSTEM_PROMPTの直近イベントをClaudeが告知文に変換して全員に配信
/告知 [内容]  → 指定した内容の告知文をClaudeが生成して全員に配信（例: /告知 スポーツ大会）
/全員 [MSG]   → 自由なテキストをそのまま全員に一斉配信
/登録者       → Google Sheetsに登録されているメンバー数を表示
/イベント     → SYSTEM_PROMPTに書かれた今後のイベント一覧を確認（管理者のみ）
/ヘルプ       → コマンド一覧を表示
タグ配信について： 特定イベント参加者だけに絞り込んで配信する「タグ配信」は、LINE OA Manager（manager.line.biz）のブラウザ画面から手動で行います。BotはタグのAPIを使わず、OA Managerのタグ機能を活用します。

5. イベント情報の更新方法 ― 最重要！

⚠️ 重要： Botが返すイベント情報・AI応答の内容はすべて SYSTEM_PROMPT という環境変数から来ています。架空のイベントを生成しないよう設計されており、ここを更新するだけで全機能に反映されます。

SYSTEM_PROMPTの更新手順

1
Railway（railway.com）にログインして earnest-surprise プロジェクトを開く
2
OGM-official サービスをクリック → 上部タブの Variables をクリック
3
SYSTEM_PROMPT の行の右端 ⋮（三点メニュー） → Edit をクリック
4
テキストを編集（イベント情報を追加・変更・削除）して ✓ チェックマーク で保存
5
保存後に Deploy ボタンを押すと新しい設定でBotが再起動する
📌 SYSTEM_PROMPTに書くべき内容
  • OGMの団体概要・ミッション・活動内容
  • 今後のイベント一覧（日付・場所・定員・LINEグループリンクなど）
  • 申込方法・注意事項
  • AIがどんなトーンで返答すべきかの指示
  • 過去のイベントはレコメンドしないなどのルール

7. 環境変数一覧（Railway Variables）

Botの動作に必要な設定値はすべてRailwayの「Variables」画面で管理されています。コードに直接書かないことでセキュリティを保っています。

変数名用途変更が必要なとき
TOKENLINE Messaging API のアクセストークン。LINEへのメッセージ送信に使用。LINEのトークンを再発行したとき
SECRETLINE チャンネルシークレット。Webhookの正当性確認に使用。LINEのシークレットが変わったとき
ANTHROPIC_API_KEYClaude AI（Anthropic）のAPIキー。AI応答・告知文生成に使用。APIキーを再発行したとき
SYSTEM_PROMPTClaudeへのシステムプロンプトOGMの説明・イベント情報・応答ルールなどを記載。最もよく更新する変数。イベント情報を更新するたびに変更
GREETING_MESSAGE友だち追加時に自動送信されるメッセージ。挨拶文を変えたいとき
ADMIN_IDS管理者コマンドを使えるLINEユーザーIDリスト（カンマ区切り）。管理者を追加・変更するとき
SPREADSHEET_IDGoogle SheetsのID。登録データの保存先を指定。スプレッドシートを変更したとき
GOOGLE_ACCESS_TOKENGoogle Sheets APIの認証トークン。現在未設定のため、シート書き込みは待機中。GCPサービスアカウントを設定するとき
PORTサーバーのポート番号（Railwayが自動管理）。通常変更不要
⚠️ Google Sheetsへの書き込みについて： 現在 GOOGLE_ACCESS_TOKEN が未設定のため、メンバー登録データの自動保存は動作しません。Google Cloud Platformでサービスアカウントを作成し、トークンを設定することで有効になります。

8. よくある質問 / トラブルシューティング

❓ Botが返事をしない
  • RailwayでサービスがOnlineになっているか確認
  • LINE DevelopersのWebhook URLが正しいか確認
  • Railway Deploymentsのログを確認
❓ 管理者コマンドが効かない
  • 自分のLINE User IDがADMIN_IDSに入っているか確認
  • LINE User IDはLINE Developers → Botとのチャットで確認可能
❓ 告知文がおかしい内容になる
  • Railway → SYSTEM_PROMPTのイベント情報を最新に更新する
  • 過去のイベントが残っていないか確認して削除する
❓ コードを変えたのに反映されない
  • GitHubにコミット済みか確認
  • RailwayのDeploymentsでビルド成功しているか確認
  • 手動でDeployボタンを押してみる
